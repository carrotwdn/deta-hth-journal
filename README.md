# deta-hth-journal

## 简介

`deta-hth-journal` 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定的域名或网站，主要用途是保存一些独立的、可离线查看的网页内容，方便日后查阅与分享。

这些页面可能包含个人笔记、临时文档、实验性页面或其他独立内容，均以纯 HTML 文件的形式存放。

## 目录说明

```
/
├── index.html          # 仓库首页（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── example-1.html
│   └── example-2.html
├── assets/             # 静态资源（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有归档的 HTML 页面，文件名应简洁明了。
- `assets/`：存放页面所需的附属资源，按类型分目录管理。
- `index.html`（可选）：可放置一个简单的导航或说明页面。

## 页面归档说明

本仓库中的 HTML 页面均为独立文件，不依赖外部服务或动态后端。每个页面应尽量自包含，如需引用外部资源（如 CSS、JavaScript、图片），建议将资源文件一并放入 `assets/` 目录中，并使用相对路径引用。

页面内容可以是任何形式的静态网页，包括但不限于：

- 个人笔记或日记
- 临时项目展示
- 学习实验页面
- 文档快照

请勿将本仓库用于托管任何违反法律法规或平台政策的内容。

## 维护说明

- 新增页面：直接在 `pages/` 下创建新的 HTML 文件，并确保引用资源路径正确。
- 更新页面：覆盖原有文件或新建版本文件（建议在文件名中加入日期或版本号）。
- 删除页面：直接移除文件，并检查是否有不再使用的资源文件需要一并清理。
- 资源管理：若页面不再使用某资源，请及时从 `assets/` 中删除，保持仓库整洁。
